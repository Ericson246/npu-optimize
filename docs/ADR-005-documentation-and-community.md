# ADR-005: Documentation and Community

**Date:** 2026-06-14
**Status:** Accepted
**Repo:** `github.com/Ericson246/npu-optimize`

## Context

`npu-optimize` is an open-source project that needs a professional presence from day 1 to attract contributors, users, and facilitate maintenance.

Based on analysis of reference projects (llama.cpp, Ollama, goreleaser), the files, policies, and documentation structure are defined below.

### CONTRIBUTING.md

Inspired by llama.cpp (AI policy) + goreleaser (specific commands, conventions).

Sections:

1. **Code of conduct** — refer to `.github/CODE_OF_CONDUCT.md`
2. **Setup** — `go install`, clone, `go mod tidy`, `go build ./cmd/npu-optimize`
3. **Tests**:
   ```bash
   go test ./... -v -count=1                # Unit tests
   go test -tags=integration ./...          # Integration (opt-in)
   golangci-lint run                        # Linting
   go build ./cmd/npu-optimize              # Verify compilation
   ```
4. **Commit convention** — Conventional Commits:
   ```
   feat: add --output-schema-version flag
   fix: error parsing GGUF header with MoE
   docs: update ADR-001 with backend interface
   test: add tests for extrapolation formula
   ```
5. **PR process**:
   - Fork + branch
   - CI must pass (lint + test + build)
   - Add tests for new code
   - Update ADRs if architecture changes
   - One change per PR
6. **AI policy**:
   - AI tools allowed but must be **declared** in the PR
   - Method: `Co-authored-by: tool-name <tool@ai>` in commit or `Assisted-by:` in description
   - Contributor must **understand and review** every generated line
   - PRs entirely generated by AI without human review will be closed
7. **Project structure** — link to ADR-001
8. **Reporting bugs** — link to issue template
9. **Feature requests** — direct to GitHub Discussions

### CHANGELOG.md

Automated from v0.1.0 with `git-chglog`. Configuration in `.chglog/`:

```yaml
# .chglog/config.yml
style: github
template: CHANGELOG.tpl.md
info:
  title: CHANGELOG
  repository_url: https://github.com/Ericson246/npu-optimize
options:
  commits:
    filters:
      Type:
        - feat
        - fix
        - docs
        - test
        - refactor
  commit_groups:
    title_maps:
      feat: Added
      fix: Fixed
      docs: Documentation
      test: Tests
      refactor: Refactored
  header:
    pattern: "^(\\w*)(?:\\(([\\w\\$\\.\\-\\*\\s]*)\\))?\\:\\s(.*)$"
    pattern_maps:
      - Type
      - Scope
      - Subject
```

Generated automatically in the release CI job (`.github/workflows/release.yml`):
```bash
git-chglog --next-tag ${{ github.ref_name }} -o CHANGELOG.md
```

### Governance Model

| Role | Who | Responsibilities |
|:-----|:----|:-----------------|
| **BDFL** | Ericson246 | Final decisions on architecture, roadmap, releases. Veto power in conflict cases. |
| **Maintainers** | Appointed by BDFL | PR review, issue triage, contributor guidance |
| **Contributors** | Anyone | PRs following CONTRIBUTING.md |

Decision process:
- **Minor changes** (bug fixes, tests, docs): Any maintainer can approve
- **Significant changes** (new features, API changes): 48h lazy consensus + approval from at least 1 maintainer
- **Architectural changes** (new ADRs, breaking changes): Requires ADR + BDFL approval
- **Disputes**: Escalated to BDFL. If BDFL is unavailable, the dispute is documented and awaits resolution.

### Community Policies

| Aspect | Decision |
|:-------|:---------|
| **Issues** | Bugs only. Features → Discussions |
| **PRs** | Require green CI + completed checklist |
| **Commits** | Conventional Commits (`feat:`, `fix:`, `docs:`, `test:`, `refactor:`) |
| **Releases** | Semver (`v0.1.0`, `v0.2.0`, ... → `v1.0.0`) |
| **AI policy** | Allowed with mandatory declaration and human review |
| **Language** | Bilingual README (English + Spanish). ADRs in English (from v0.1.0). Issues and PRs in Spanish. |
| **Discussion channel** | GitHub Discussions (no Discord, no Telegram for now) |
| **Governance** | BDFL + lazy consensus for contributions |

## Alternatives Considered

| Alternative | Discarded because |
|:------------|:------------------|
| GitHub Wiki for docs | ADRs should be versioned with code; wiki is not |
| mdbook / docsify | Additional maintenance. Plain markdown ADRs are sufficient for v0.x |
| Discord / Telegram | Maintenance and moderation overhead. Discussions is sufficient for initial scale |
| README only in Spanish | Chose bilingual README from v0.1.0 to avoid excluding international contributors |
| `AGENTS.md` in `.github/` | More visible at root (Ollama pattern) |
