# Contributing to npu-optimize

## Code of Conduct

This project follows the [Contributor Covenant v2.1](.github/CODE_OF_CONDUCT.md).

## Setup

```bash
git clone https://github.com/Ericson246/npu-optimize
cd npu-optimize
go mod tidy
go build ./cmd/npu-optimize
```

## Tests

```bash
go test ./... -v -count=1          # Unit tests
go test -tags=integration ./...    # Integration (opt-in)
golangci-lint run                  # Linting
go build ./cmd/npu-optimize        # Verify compilation
```

## Commit Convention

We use [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add --output-schema-version flag
fix: error parsing GGUF header with MoE
docs: update ADR-001 with backend interface
test: add tests for extrapolation formula
```

## PR Process

1. Fork + branch
2. CI must pass (lint + test + build)
3. Add tests for new code
4. Update ADR-007 if architecture changes
5. One change per PR

## AI Policy

- The use of AI tools is allowed but must be declared in the PR
- Method: `Co-authored-by: tool-name <tool@ai>` in the commit or `Assisted-by:` in the description
- The contributor must understand and review every generated line
- PRs entirely generated by AI without human review will be closed
